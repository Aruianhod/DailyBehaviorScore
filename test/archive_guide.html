📁 学生行为分管理系统 - 归档功能使用说明

完整的归档数据管理解决方案

📋 功能概述

归档功能用于将已毕业学生的完整数据进行备份并从活跃数据库中删除，以保持系统性能和数据整洁。

💾

数据备份

完整保存学生信息、分数记录和申请历史

🗑️

数据清理

安全删除活跃数据库中的历史数据

📊

统计信息

提供详细的归档统计和日志记录

⬇️

文件下载

支持JSON格式的归档文件下载

⚡ 归档条件

系统自动判断哪些年级可以被归档：

归档条件： 年级 + 4 ≤ 当前年份
示例： 2025年可以归档2021年及以前入学的学生
⚠️ 注意： 只有满足毕业条件的年级才会在归档列表中显示

📝 操作步骤

登录管理员账户

使用管理员权限登录系统，进入归档管理页面

选择归档年级

系统会自动显示可归档的年级列表，选择需要归档的年级

填写归档原因

输入归档原因（如"学生已毕业4年，进行数据归档"）

执行归档操作

系统会自动备份数据并删除原始记录

下载归档文件

在归档历史中查看并下载归档文件

📄 归档文件格式

归档文件采用JSON格式，包含以下结构：

{ "archiveInfo": { "id": 25, "archiveDate": "2025-06-02", "grades": ["2018", "2019"], "reason": "学生已毕业，进行数据归档", "createdBy": "admin", "createdAt": "2025-06-02T08:30:15.000Z" }, "statistics": { "studentCount": 150, "recordCount": 450, "applicationCount": 75 }, "students": [ { "student_id": "20180001", "name": "张三", "grade": "2018", "class": "1班", "score": 95, "records": [...], "applications": [...] } ] }
📌 说明：
  • archiveInfo：归档基本信息和元数据
  • statistics：归档数据的统计信息
  • students：完整的学生数据，包括分数记录和申请历史

🔄 数据恢复

如需恢复归档数据，可以：

下载归档文件

从归档历史中下载对应的JSON文件

解析JSON数据

使用JSON解析工具或编程语言处理文件内容

重新导入数据

根据需要将学生数据重新导入到系统中

⚠️ 重要： 数据恢复操作需要技术人员协助，请谨慎操作

🔧 技术说明

API 接口

  • GET /api/admin/archive/logs - 获取归档历史记录
  • GET /api/admin/archive/download/:id - 下载归档文件
  • POST /api/admin/archive/execute - 执行归档操作
数据库表：
  • archive_logs - 存储归档日志和元数据
  • students - 学生基本信息
  • score_records - 分数变动记录
  • score_applications - 分数申请记录
